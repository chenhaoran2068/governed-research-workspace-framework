# Public And Private Boundary

## Public Framework

The public framework may publish generic architecture, schemas, templates,
synthetic examples, installation contracts, and validation guidance.

## Private Workspace

A private workspace may contain real projects, local configuration, approved
internal connectors, restricted sources, unpublished outputs, detailed audit
records, and accumulated experience.

## One-Way Derivation

```text
private source
  -> generic rewrite and review
  -> public framework or system release
```

The reverse direction is prohibited: a public repository must not overwrite or
become authority for private workspace materials.

## Source And Rights Control

Source pointers, metadata, and independently authored summaries may be public
when rights permit. Full source payloads, licensed material, private notes, and
project claim audits remain excluded unless separately reviewed.
