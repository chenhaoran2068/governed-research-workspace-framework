# Contributing

Contributions must preserve the framework's generic, path-independent, and
public-safe scope.

Before proposing a change:

1. identify the contract or schema affected;
2. state whether the change is backward compatible;
3. include or update a synthetic example and validation coverage;
4. confirm that no real project, private path, restricted source, credential,
   or unpublished material is included; and
5. avoid claiming profile support without demonstrated tests.

Do not submit real study files, patient-derived data, manuscript text, review
correspondence, or credentials.
