# Security Policy

Do not report security concerns through public issues when they could expose
credentials, private paths, restricted data, or a practical method to bypass a
framework boundary.

Until a public repository and maintainer contact are established, report a
potential issue privately to the repository maintainer through the hosting
platform's private contact channel. Do not include sensitive material in the
initial report.

This framework provides architectural contracts, not a security boundary by
itself. Host permissions, source-system access controls, and human approval
remain independently required.
